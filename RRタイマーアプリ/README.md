# ポーカートーナメントタイマーアプリ

## 概要

これはテキサスホールデム（ポーカー）のトーナメントで使用するための専用タイマーアプリです。大きな画面に表示しながら、スマートフォンやタブレットなどの別端末で操作が可能です。

## 主な機能

### 🎯 基本機能
- **複数タイマー管理**: 異なるトーナメント形式に対応した複数のタイマーを作成・管理
- **ブラインドレベル管理**: 各レベルごとのSB（スモールブラインド）、BB（ビッグブラインド）、ANTEの設定
- **ブレイクタイマー**: レベル間の休憩時間を自動管理
- **次レベル表示**: 次のブラインドレベルと次のブレイクまでの時間を表示

### 📊 情報表示
- **トーナメント名**: 動的に変更可能
- **タイマー表示**: 大きな文字で残り時間を表示（1分未満で黄色に変化）
- **現在のブラインド**: SB、BB、ANTEを明確に表示
- **次のブラインド**: 次のレベルのブラインド情報を表示

### 🎁 プライズ管理
- **REWARD表示**: 入賞賞金や景品の表示
- **プライズプール計算**: 参加費×受付人数で自動計算
- **アベレージスタック**: 現在の平均チップ量をリアルタイム表示

### 👥 参加者管理
- **参加人数追跡**: 現在参加者数と受付人数（エントリー+リエントリー）を表示
- **リアルタイム更新**: 管理画面からの変更が即座に反映

### 🎵 音楽通知
- **1分前**: 「ピーン」という軽いポップな音
- **レベル変更時**: 「ポーンポーン」という明るい音楽的な音

### 🎯 ブレイク設定
- **任意のレベル後に設定**: 特定のレベル終了後にのみブレイクを設定可能
- **ブレイクカット機能**: ブレイク中に30秒に強制短縮

### 🎮 コントロール機能（管理画面）
- **前のレベル/次のレベル**: 手動でレベル移動
- **スタート/ストップ**: タイマーの開始と停止
- **ブレイクカット**: ブレイク中に押すと残り30秒に強制変更
- **更新ボタン**: 編集内容を即座に反映

## 画面構成

### メイン画面（index.html）
- **左8割**: タイマー、トーナメント名、現在のブラインド、次レベル情報
- **右2割**: REWARD表示、アベレージスタック、参加人数比、コントロールボタン

### 管理画面（admin.html）
- **基本情報設定**: トーナメント名、参加人数、初期スタック
- **タイマー管理**: 複数タイマーの作成・編集、レベル設定
- **ブレイク設定**: 休憩時間の設定
- **プライズ設定**: 賞金・景品の設定
- **プライズプール**: 参加費と受付人数から自動計算

## 使い方

### 初期設定
1. 管理画面（admin.html）を開く
2. トーナメント名、参加費、初期スタックを設定
3. タイマーを作成し、各レベルのブラインドと時間を設定
4. 必要に応じてブレイクを設定
5. プライズを設定
6. 保存ボタンをクリック

### タイマー操作
- **管理画面で操作**: スタート/ストップ、前のレベル/次のレベル、ブレイクカット
- **キーボードショートカット**: スペースキーで開始/停止、矢印キーでレベル移動、Rキーでリセット

### デモモード
- 初期状態でデモデータが自動設定（2分間の短いレベル）

### リアルタイム同期
- 管理画面での変更がメイン画面に自動反映（5秒間隔で更新）
- 複数のデバイスで同時に管理可能

## 技術仕様

### フロントエンド技術
- HTML5、CSS3、JavaScript（ES6）
- レスポンシブデザイン
- Web Audio API（音楽通知）
- LocalStorage（データ永続化）

### デザイン
- 背景: スカイブルーのグラデーション
- フォント: ゴシック体（日本語対応）
- 配色: 白文字、黄色の警告表示、金色の強調

### データ管理
- ブラウザのLocalStorageを使用したデータ保存
- リアルタイムデータ同期（5秒間隔での自動更新）

## キーボードショートカット

| キー | 機能 |
|------|------|
| スペース | タイマー開始/停止 |
| ← | 前のレベル |
| → | 次のレベル |
| R | タイマーリセット |

## 注意事項

- このアプリはブラウザで動作する静的ウェブアプリケーションです
- サーバーとの通信は行わず、すべてブラウザ内で動作します
- データはブラウザのLocalStorageに保存されます
- プライベートモードではデータが保持されない場合があります
